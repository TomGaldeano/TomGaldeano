Theory

  • git uses name and email to lable the commits
  • Use annotated tags for anything public or shared
  • Use tags after passing all test or before release and to mark releases or major milestones
  • git stash allows to save uncommited changes to return to a clean directory
  • Each time you run git stash, your changes are saved on top of a "stack". The most recent stash is on top, and you can apply or drop stashes from the top down, or pick a specific one from the list.
  • A merge conflict happens when changes in two branches touch the same part of a file and Git doesn't know which version to keep. both willbe kept and shown with extra text to let you decide what to do
  • Exclude files that shouldn't be tracked (like build artifacts, log files, or secrets) by adding them to a .gitignore file.
  • The .gitignore file is used to specify which files git should not keep track of
  • The .gitattributes file is a special file that tells Git how to handle specific files in your repository.
  • Git LFS (Large File Storage) is an extension for Git that helps you manage large files (like videos, images, or datasets) efficiently. Instead of storing big files directly in your repository, LFS stores a small pointer file in your repo and keeps the real content on a separate LFS server.
  • Signing a commit is like putting your personal signature on your work.
  • Git hooks are scripts that run automatically when certain Git events happen, like making a commit or pushing code.

Statements and keywords

  • git help command shows help for the command
  • git config --local user.email="email" sets the users email. To change a value run it again with new value
  • git config --global user.name="name" changes the users name.
  • git config --list lists all settings
  • git config user.name shows specified setting
  • git config --global --unset code.editor removes a setting
    • --golbal does it for every repo of that user
    • --local is the default so it is not needed and sets for the local rep
    • --system sets for every user and repo in the system
  • git init initializes a git repo in the current folder
  • git status shows the status of the repo
  • git status -s shows the status of the repo in short
  • git status -b shows the status of the repo with the branch
  • git add file adds file or files to the staging area
  • git add . git add -A git add -all adds all files to the staging area

    • Git commmit

  • git commit -m "message" commits the staged files with a message
  • git commit -a -m "message" adds all files and commits them with a message. does not commit untracked(new)files
  • git commit --amend -m "new message" changes the message of the last commit
  • git commit --no-edit uses preevious commit message
  • git commit --amend --no-edit adds changes to last commit keeping the message

    • History: got log, git show, git diff

  • git log shows the commit history
  • git log --oneline shows the commit history in one line per commit
  • git log --stat shows the commit history with stats
  • git log -p shows the commit history with the changes made
  • git log --pretty=format:"%h - %an, %ar : %s" shows the commit history with a custom format
  • git log --graph shows a simple ascii graph of the commit history
  • git log --all shows the commit history of all branches
  • git log --decorate shows the commit history with the branch and tag names
  • git log --graph --oneline --all --decorate shows a graph of the commit history
  • git log --since="2 weeks ago" shows the commit history since 2 weeks ago
  • git log --after="2024-01-01" shows the commit history after the date
  • git log --before="2024-01-01" shows the commit history before the date
  • git log --author="name"shows logs made by that g
  • git diff shows the changes made but not staged
  • git diff --staged shows the changes staged
  • git diff commit_id1 commit_id2 shows the changes between the two commits
  • git show commit_id shows the commit id details
  • git show branch_name shows the branch details
  • git show tagname shows the tag details
  • git show HEAD shows the last commit details
  • git show HEAD~1 shows the commit before the last commit details

    • Git branches

  • git branch lists all branches
  • git branch branch_name creates a new branch
  • git checkout branch_name switches to the branch
  • git checkout -b branch-namecreates and checksout branch-name
  • git merge branch_name merges the branch into the current branch
  • git merge --no-ff branch always creates a merge commit
  • git merge --squash combines merges into a single commit
  • git merge --abort abort a merge in progress
  • git remote add origin url adds a remote repo

    • Git push

  • git push -u origin branch_name pushes the branch to the remote repo
  • git push pushes the changes to the remote repo
  • git pull pulls the changes from the remote repo
  • git fetch origin fetches the changes from the remote repo, does not modify working files but let's you see changes made before merging
  • git clone url clones a remote repo

    • Git remove

  • git rm file_name removes a file from the repo
  • git rm --cached file_name removes a file from the repo but keeps it in the local directory

    • Git stash

  • git stash stashes the changes made to have a clean directory and save the changes
  • git stash list lists the stashed changes
  • git stash show shows the changes made in the last stash
  • git stash show -p shows the changes made in the last stash with the changes
  • git stash apply applies the stashed changes but keeps them in the stash list
  • git stash push -m stashes with a message
  • git stash pop applies the stashed changes
  • git stash drop removes the last stash
  • git stash drop stash@{index} removes the stash at the index
  • git stash clear removes all stashes
  • git stash branch branch_name creates a branch from the last stash and applies the changes
  • git stash branch branch_name stash@{index} creates a branch from the stash at the index and applies the changes

    • Git reset and restore

  • git reset --hard commit_id resets the repo to the commit id
  • git reset --soft commit_id resets the repo to the commit id but keeps the changes staged
  • git revert commit_id reverts the commit id by creating a new commit without the changes. preserves commit history
  • git revert HEAD --no-edit fetches the changes from the remote repo
  • git reset --hard origin/branch_name resets the local branch to the remote branch
  • git restore file_name restores the file to the last commit version
  • git restore --staged file_name removes the file from the staging area but keeps the changes made
  • git reflog
    • Recover lost commits or changes
    • Undo a reset or a merge
    • See the history of your branch and HEAD
    • Find the commit hash of a previous state
    • View changes made in the repository over time
    • reflog is automatically cleaned by git

    Git tag

  • git tag tag_name creates a tag
  • git tag -a tag_name -m "message" creates an annotated tag with a message wich also stores name and date
  • git tag lists the tags
  • git tag -l "pattern*" lists the tags that match the pattern
  • git checkout tag_name switches to the tag in a detached head state so no changes can be made
  • git tag -f name_of_tag new_commit_hash moves the tag named to the new commit must be forced pushed after
  • push --force origin name_of_tag forcefilly pushes the tag to remove the other
  • git tag -d tag_name deletes the tag
  • git push origin --tags pushes all tags to the remote repo
  • git push origin tag_name pushes the tag to the remote repo

    • Rebase

  • git rebase branch_name rebases the current branch onto branch_name
  • git rebase --abort aborts the rebase
  • git rebase --continue continues the rebase after resolving conflicts
  • git rebase --skip skips the current commit during rebase
  • git pull --rebase pulls the changes from the remote repo and rebases
